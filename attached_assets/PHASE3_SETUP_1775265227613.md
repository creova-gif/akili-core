# AKILI PHASE 3 — Setup Instructions
## Wiring live automation into your existing Replit project

---

## What Phase 3 adds

1. **PULSE auto-scheduler** — Akili generates posts on schedule and sends
   them to Telegram for your approval before posting. Reply POST, EDIT, or SKIP.

2. **REACH auto-responder** — Reads your Gmail every 30 min, classifies emails,
   auto-replies to fans/business/collab in your voice, flags urgent ones instantly.

3. **INTEL live brief** — Your 8AM morning brief now uses real web search to
   pull live African tech news, VC moves, and competitor updates.

4. **Dashboard API** — The command bar in your AKILI dashboard now talks to
   real agents via a FastAPI server running on port 8080.

---

## Steps in Replit

### 1. Add new files to your existing project
Upload these files into your Replit (in the scheduler/ and api/ folders):

```
scheduler/
  pulse_scheduler.py      ← NEW
  reach_autoresponder.py  ← NEW
  intel_live.py           ← NEW
api/
  server.py               ← NEW
main.py                   ← REPLACE your existing main.py with this one
requirements.txt          ← REPLACE with this updated version
```

### 2. Create the folders in Replit
In Replit shell:
```
mkdir -p scheduler api
```

### 3. Add one new secret
In Replit → Secrets tab, add:
```
AKILI_API_SECRET = akili-justin-2026
```
(Pick any secret string — you'll use this in your dashboard)

### 4. Update your dashboard command bar
In your index.html, find the fireCmd() function and update it:

```javascript
async function fireCmd() {
  const v = document.getElementById('cmdInput').value.trim();
  if (!v) return;

  addFeedItem('⚙️', 'COMMAND', v, 'now');
  document.getElementById('cmdInput').value = '';

  try {
    // Replace YOUR-REPLIT-URL with your actual Replit URL
    const res = await fetch('https://YOUR-REPLIT-URL.repl.co/command', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ command: v, secret: 'akili-justin-2026' })
    });
    const data = await res.json();
    addFeedItem(
      data.agent === 'SHIELD' ? '🛡' : data.agent === 'PULSE' ? '📡' : '⚡',
      data.agent,
      data.response.slice(0, 120) + '...',
      'just now'
    );
  } catch(e) {
    addFeedItem('⚠️', 'ERROR', 'API not reachable — is Akili running?', 'now');
  }
}
```

### 5. Run Akili
Click Run in Replit. You should see:
```
AKILI OS Phase 3 — all systems running
PULSE Scheduler initialized
REACH AutoResponder initialized
INTEL LiveBrief initialized — web search enabled
```

## How to use Phase 3

### PULSE approval flow (new)
Every day at scheduled times, Akili sends you a message like:
```
📡 PULSE — Post ready for approval
Platform: TWITTER
Theme: Tech Tuesday
...
Reply: POST twitter_Tuesday_8 to approve
```

You reply:
- `POST twitter_Tuesday_8` → posts immediately
- `EDIT twitter_Tuesday_8 [your new version]` → uses your text instead
- `SKIP twitter_Tuesday_8` → skips this post

### REACH email replies (automatic)
REACH checks Gmail every 30 min silently.
Urgent emails → instant Telegram alert to you.
Fan/business/collab emails → auto-replied in your voice, you never see them.

To draft a reply yourself: `draft reply [context]`
To see what was replied: check your Gmail Sent folder.

### Live research (new)
Instead of: "research GoPay competitors"
Akili now ACTUALLY searches the web and returns live results.

Try:
- `research East Africa fintech funding 2026`
- `vc tracker` → live GoPay investor intelligence
- `competitor GoPay` → live competitor news

### /pending command
Send `/pending` to Telegram → see all posts waiting for your approval.

## Troubleshooting

**Scheduler not sending posts?**
→ Check your timezone — schedule runs in server time (UTC).
→ Adjust POST_SCHEDULE hours in pulse_scheduler.py accordingly.
  (If you're in EST = UTC-4, add 4 to every hour)

**REACH not replying to emails?**
→ Make sure gmail_credentials.json is in your config/ folder
→ Run `python integrations/gmail.py` once to authorize

**API 401 error from dashboard?**
→ Check AKILI_API_SECRET matches in both Replit Secrets and index.html

**Intel brief not searching?**
→ Web search requires Anthropic API access — check your API key has access
